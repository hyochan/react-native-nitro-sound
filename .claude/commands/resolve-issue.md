# Resolve Issue

Analyze a GitHub issue in `hyochan/react-native-nitro-sound`, add labels, and either fix it with a PR or comment with analysis.

## Usage

```text
/resolve-issue <ISSUE_NUMBER_OR_URL>
```

## Examples

```text
/resolve-issue 741
/resolve-issue https://github.com/hyochan/react-native-nitro-sound/issues/741
```

## Arguments

- `$ARGUMENTS` - Issue number (e.g., `741`) or full issue URL

## Instructions

When this command is executed, perform the following. Use `gh` CLI (or the GitHub REST API with `$GH_TOKEN`) for all GitHub operations.

### 1. Gather Issue Information

```bash
gh issue view $ARGUMENTS --repo hyochan/react-native-nitro-sound \
  --json number,title,body,state,labels,author,comments
# Related PRs
gh pr list --repo hyochan/react-native-nitro-sound \
  --search "$ISSUE_NUMBER in:body" --json number,title,state
```

### 2. Add Labels

Based on issue content, add appropriate labels:

```bash
gh issue edit $ISSUE_NUMBER --repo hyochan/react-native-nitro-sound \
  --add-label "<label1>,<label2>"
```

**Label selection guide:**

| Condition                                          | Label                          |
| -------------------------------------------------- | ------------------------------ |
| Bug report / crash                                 | `🐛 bug`                        |
| New feature request                                | `🍗 enhancement`                |
| iOS-only (mentions iOS, StoreKit, AVAudioSession)  | `📱 iOS`                        |
| Android-only (mentions Android, MediaRecorder)     | `🤖 android`                    |
| Build / Gradle / Xcode compile failure             | `👷‍♀️ build`                      |
| Needs reproduction or more info                    | `ℹ needs more info`            |
| Needs deeper investigation                         | `🕵️‍♂️ need more investigation`    |
| Community help welcome                             | `🙏 help wanted`                |
| Refers to legacy `react-native-audio-recorder-player` API only | `🗑 stale-legacy`   |
| Breaking change                                    | `⚡️ breaking`                   |
| Refactor                                           | `🝤 refactor`                  |
| Documentation                                      | `📖 documentation`              |
| CI/CD                                              | `💨 ci`                         |

### 3. Analyze and Decide

Classify the issue:

- **Fixable now** → Proceed to Step 4 (create fix PR).
- **Needs more info** → Comment asking for a minimal repro, library version, RN version, platform.
- **Feature request / discussion** → Comment with feasibility assessment.
- **Already fixed** → Comment with the commit / PR / release that addressed it and close.
- **Legacy (pre-Nitro) bug that no longer exists in `react-native-nitro-sound`** → Comment referencing the Nitro rewrite and close as not planned. Apply the `🗑 stale-legacy` label.
- **Out of scope / won't fix** → Comment explaining why and close.

### 4. Fix and Create PR (if fixable)

#### 4a. Create a fix branch

```bash
git checkout main && git pull
git checkout -b fix/<scope>-<short-description>
```

**Scope naming:**

- `ios` — iOS / Swift code under `ios/`
- `android` — Android / Kotlin code under `android/`
- `ts` / `js` — TypeScript in `src/`
- `nitro` — Changes involving Nitrogen-generated specs or `.nitro.ts`
- `example` — Example app under `example/`
- `ci` — GitHub Actions / workflows
- `docs` — README / docs only

#### 4b. Investigate and fix

1. Read the code paths mentioned in the issue.
2. Understand the root cause.
3. Implement the fix.
4. Run build/verify commands for the area touched:

| Area touched             | Commands                                                                  |
| ------------------------ | ------------------------------------------------------------------------- |
| TypeScript (`src/`)      | `yarn typecheck && yarn lint`                                             |
| iOS (`ios/`)             | `cd example/ios && pod install && xcodebuild -workspace NitroSoundExample.xcworkspace -scheme NitroSoundExample -configuration Debug -sdk iphonesimulator build` |
| Android (`android/`)     | `cd example/android && ./gradlew :app:assembleDebug`                      |
| Nitrogen specs           | `yarn specs` (regenerate nitrogen) then run iOS & Android builds          |

#### 4c. Commit and push

```bash
git add <files>
git commit -m "fix(<scope>): <description>

Closes #$ISSUE_NUMBER

🤖 Autogenerated by Claude — react-native-nitro-sound is an AI-native
maintained repo. See README for how this works.
Co-Authored-By: Claude <noreply@anthropic.com>"
git push -u origin <branch-name>
```

#### 4d. Create PR

```bash
gh pr create --title "fix(<scope>): <description>" --body "$(cat <<'EOF'
## Summary
<1-3 bullet points>

Closes #$ISSUE_NUMBER

## Test plan
- [ ] Type check passes
- [ ] iOS example builds
- [ ] Android example builds

🤖 Autogenerated by Claude (AI-native maintenance).
EOF
)"
```

#### 4e. Mirror labels onto the PR

```bash
gh pr edit <PR_NUMBER> --repo hyochan/react-native-nitro-sound \
  --add-label "<label1>,<label2>"
```

> If `gh pr edit --add-label` fails with a Projects GraphQL error, use REST directly:
>
> ```bash
> gh api -X POST repos/hyochan/react-native-nitro-sound/issues/<PR_NUMBER>/labels \
>   -f "labels[]=<label1>" -f "labels[]=<label2>"
> ```

### 5. Comment on the Issue

Always leave a comment summarizing what you did.

**If fixed with PR:**

```bash
gh issue comment $ISSUE_NUMBER --repo hyochan/react-native-nitro-sound --body "$(cat <<'EOF'
<Root cause analysis in 1–3 sentences.>

Fix is up in #<PR_NUMBER>.

> **Note:** This repo is maintained by Claude as an AI-native project. Hyo triages direction; fixes and dep bumps are autogenerated. See the README for details.
EOF
)"
```

**If needs more info:**

```bash
gh issue comment $ISSUE_NUMBER --repo hyochan/react-native-nitro-sound \
  --body "<Specific questions, requested info: lib version, RN version, platform, minimal repro.>"
```

**If legacy / stale:**

```bash
gh issue comment $ISSUE_NUMBER --repo hyochan/react-native-nitro-sound --body "$(cat <<'EOF'
This issue refers to behavior in the pre-Nitro `react-native-audio-recorder-player` API, which has been rewritten on top of react-native-nitro-modules. If this still reproduces on the latest `react-native-nitro-sound`, please open a fresh issue with a minimal repro and version info — closing as `stale-legacy` for now.
EOF
)"
gh issue close $ISSUE_NUMBER --repo hyochan/react-native-nitro-sound --reason "not planned"
```

### 6. Report Back

Summarize:

- Issue classification
- Labels added
- Action taken (PR created / comment posted / closed)
- PR link (if created)
